PostgreSQL

Components

Postgres comes as a package with several components:

The Database Server
The database server runs as service or plain executable and communicates over Unix domain sockets or IP sockets.
pg_ctl.exe
A utility to initialize and control a PostgreSQL server. To start the server, run the following command: 
D:\progs\pg\bin\pg_ctl.exe" start -D "D:\progs\pg\data"
The -D argument specifies the data directory and is mandatory.
On Windows, the Postgres server can be started as a service: 
"D:\progs\pg\bin\pg_ctl.exe" runservice -N "postgresql-x64-14" -D "D:\progs\pg\data" -w
The service name may have to be adjusted to your system.
Documentation: www.postgresql.org
psql.exe
This is the Interactive Postgres terminal, used for database maintenance, configuration, data retrieval, data import and export, and scripting. This workhorse will be used extensively in the tutorials.
Documentation: www.postgresql.org
pgadmin
Postgres database administration GUI. Not covered in this tutorial; I strongly recommend to learn the concepts and SQL commands with psql first, otherwise you could easily get overwhelmed by the complexity of this tool.
Client Interfaces and Drivers
ODBC, JDBC drivers and .NET adapters are available, as well as libraries for C, C++, TCL, Go, Python, ECPG for embedding SQL in C source code, JavaScript drivers for Node and Deno et cetera. More information at postgresql.org.

Installation

Download and run the installer package from www.postgresql.org. To follow this tutorial, select the PostgreSQL Server and Command Line Tools. PgAdmin and Stack Builder are not required yet. Set the installation and data directories, and the server's socket port number. Then select the locale that suits your preferences. If you are unsure, use the default value.

A few minutes later you should have a running Postgres server running on your system.

NOTE that you can install server and client tools on different machines. This makes sense in a production environment.
