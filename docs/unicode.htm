Multilingual Support

What worked:

  • Windows console codepage set to 1252 (chcp 1252).
  • PSQL encoding set to WIN1252 (\encoding win1252).
  • Database created with UTF-8 encoding (create dabasbase dbname encoding 'UTF8').
  • CSV import file saved with encoding UTF-8 with LE BOM (byte-order-mark).
  • CSV import specified with UTF-8 encoding (COPY table-name ( field-name, ... , field-name ) from 'datafile-name.csv' with (delimiter ',' , format csv, header true, encoding 'utf8' )).

Finding: Codepage 65001 is Unicode-16 LE with BOM.
