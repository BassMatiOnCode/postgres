Data Types

Array

Postgres knows multi-dimensional arrays. In a variable declaration, each dimension is represented by a set of square brackets:

chessboard text [ ] [ ];
numbers integer ARRAY [ 4 ];
letters char ARRAY;

You can specify dimension sizes, but they are currently (v14) ignored. The standards-compliant ARRAY keyword for one-dimensional arrays may be omitted in Postgres.

Array literals are written as strings with curly braces, elements are usually separated with commas. Alternatively, the array constructor syntax can be used:

'{ 1, 2, 3 }'
'{ 1, 3, 4 } , { 4, 5 6 } , { 7, 8, 9 } }'
'{ "a" , "b" , "c" }'
ARRAY [ [ 1, 2, 3 ] , [ 4, 5, 6 ] , [ 7, 8, 9 ] ]
ARRAY [ 'a' , 'b' , 'c' ]

Values are ordinary constants or expressions. String values in array literal syntax must be enclosed in double quotes. In array constructor notation, use single quotes.

Here is an example of arrays in an SQL statement:

INSERT INTO mytable 
VALUES ( 'a', ARRAY [ 1, 2, 3 ], ARRAY [ [ 'a', 'b', 'c' ], [ 'd', 'e', 'f' ] ] ) ;

Array index values start from 1 for the first element. Slices are defined by two numbers, separated by a colon:

numbers [ 2 : 3 ]

Array Functions

array_dims
Returns the array dimensions in readable form.
array_lower, array_upper
Lower and upper bounds of an array
array_length
Number of elements in an array
array_prepend
Inserts elements in front of the array
array_append
Appends elements to the end of the array
array_cat
Concatenates arrays.

More information: postgresql.org/docs/current/arrays.
